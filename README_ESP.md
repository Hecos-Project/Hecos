# 🤖 Hecos
**Helping Companion System (Privado, Rápido, Simple)**[cite: 1]

---

> **Estado Runtime Alpha**: Hecos se encuentra actualmente en `v0.40.0`[cite: 1]. Este es un Helping Companion System local que conecta el razonamiento de alto nivel con la ejecución del sistema a nivel root[cite: 1].

## 🚀 Resumen General
**Hecos** es un **Helping Companion System**: un ecosistema local de primera línea diseñado para conectar de manera fluida la tecnología con la vida humana, combinando el razonamiento de IA, la automatización visual y la ejecución directa del sistema[cite: 1]. En lugar de perseguir conceptos abstractos como la soberanía digital, Hecos se enfoca en una única misión pragmática: **mejorar la vida humana**, transformando el hardware local en una herramienta práctica y altamente eficiente para el día a día[cite: 1].

Basado en tres pilares fundamentales:
* 🛡️ **Privacidad Primero** — Funcionamiento 100% local, cero dependencia de la nube y arquitectura de privacidad de 3 niveles[cite: 1].
* ⚡ **Velocidad Extrema** — Arquitectura nativa optimizada y sistema de complementos de alto rendimiento para una respuesta instantánea[cite: 1].
* 🧊 **Simplicidad Total** — Tablero profesional y diseño modular que hace intuitiva la orquestación de IA avanzada[cite: 1].

Ahora completamente migrado a una **arquitectura estable de Runtime Alpha**, Hecos 0.40.0 ofrece una interfaz Web dedicada (Chat + Config) e internacionalización completa[cite: 1]. Gracias a **LiteLLM**, soporta Ollama, KoboldCpp y los principales proveedores de la nube con streaming en tiempo real y TTS local[cite: 1].

## ✨ Características Principales (v0.40.0)
* ⚡ **HDCS (Comandos Directos)** — Ejecuta instantáneamente más de 150 funciones nativas omitiendo el "cerebro" de la IA escribiendo `/` en el chat o con `Ctrl+Alt+Espacio` globalmente[cite: 1].
* ⚙️ **Motor de Automatización Flows** — Editor visual de nodos (drag-and-drop) para la creación de automatizaciones complejas de múltiples pasos, activadores y acciones con integración completa de comandos de voz por NLP[cite: 1].
* **📅 Calendario Integrado** — Módulo de calendario completo con seguimiento de días festivos y codificación de colores localizada para los eventos[cite: 1].
* **⏰ Módulo Recordatorio** — Programador de alarmas y temporizadores impulsado por NLP con funcionalidades de repetición (snooze) y notificaciones activas del SO[cite: 1].
* **💻 Ventana de Acciones (Action Window)** — Consola limpia de estilo terminal directamente dentro de la interfaz de usuario de chat para monitorizar la ejecución nativa del sistema y los procesos en segundo plano[cite: 1].
* **🎵 Nuevo Reproductor Multimedia** — Backend de audio avanzado (VLC 64-bit + fallback FFplay) que soporta un estado de reanudación/pausa resiliente, cálculos dinámicos de listas de reproducción y control de volumen global[cite: 1].
* 🎨 **Flux Prompt Studio** — Ingeniería de prompts en tiempo real para Flux.1 con persistencia automática de metadatos sidecar[cite: 1].
* 🖼️ **Inyección de Metadatos en Imágenes (Image Metadata Injection)** — Los resultados de la IA generativa ahora incluyen archivos sidecar JSON ocultos (.txt) que contienen el prompt, la semilla (seed) e información del sampler para flujos de trabajo profesionales[cite: 1].
* 🎭 **Chat UI Mejorada** — Nuevos encabezados de chat con nombres de Usuario/Persona visibles, marcas de tiempo y un posicionamiento mejorado de las acciones del mensaje (Copiar/Editar/Regenerar)[cite: 1].
* 🔄 **Regeneración Corregida** — Se resolvieron los problemas críticos de duplicación de historial y errores de sesión durante la regeneración de mensajes[cite: 1].
* 🗃️ **Archivo de Chat de Modo Dual** — Sistema de archivo contextual con recuperación de chats individuales y funcionalidad de borrado masivo[cite: 1].
* 🧠 **RAG de Alto Rendimiento (FastEmbed)** — Memoria vectorial multiusuario nativa de CPU que utiliza ONNX y LanceDB para una ingesta ultra rápida de documentos[cite: 1].
* 🔐 **Aislamiento de Vault por Usuario** — Arquitectura de memoria consolidada que garantiza una separación absoluta de la privacidad de los datos semánticos y el historial de chat por cada perfil[cite: 1].
* 🛡️ **Arquitectura de Privacidad de 3 Niveles** — Gestión unificada de sesiones con los modos **Normal**, **Auto-Wipe** (almacenamiento solo en RAM, borrado automático al salir) e **Incógnito** (modalidad fantasma sin dejar rastro)[cite: 1].
* 📦 **Hecos Package Manager (HPM)** — El componente definitivo para la extensibilidad universal. Un instalador dinámico y centralizado que soporta paquetes independientes `.hpkg`. Instala fácilmente complementos de terceros, widgets y paneles de configuración con arrastrar y soltar, configuraciones aisladas y firmas digitales Ed25519[cite: 1].
* 🔌 **Universal Tool Hub (MCP Bridge)** — Soporte nativo para el **Model Context Protocol**. Conéctate a miles de herramientas de IA externas con un solo clic[cite: 1].
* 🔭 **Deep MCP Discovery** — Explorador avanzado con búsqueda multi-registro (Smithery, MCPSkills, GitHub) e instalación inmediata[cite: 1].
* 🔒 **Hecos PKI Profesional (HTTPS)** — Certificación Root CA autoafirmada (self-signed) integrada para habilitar una experiencia de "Green Lock" (candado verde) segura en todos los dispositivos de la LAN[cite: 1].
* 🏗️ **Plugin WebUI Nativo** — Interfaz de alto rendimiento optimizada y de baja latencia para escritorio y dispositivos móviles[cite: 1].
* 🎛️ **Control Room & Widgets** — Un tablero personalizable estilo masonry para widgets en tiempo real y telemetría del SO[cite: 1].
* 🌐 **Automatización del Navegador** — Plugin nativo para la interacción semántica web y el scraping[cite: 1].
* ⌨️ **Automatización del SO** — Plugin nativo para el control programático del ratón y el teclado[cite: 1].
* 💾 **Hecos Drive (File Manager)** — Gestión de archivos y editor integrado con interfaz de doble panel[cite: 1].

## 🧠 Cómo Funciona
Hecos está construido alrededor de una estructura altamente modular de 8 niveles. Todo es un **módulo**[cite: 1]:
* **Core Modules** → Funciones integradas del sistema y del SO no extraíbles[cite: 1].
* **Plugins** → Herramientas reactivas y capacidades llamadas por la IA (sistema, web, multimedia, etc.)[cite: 1].
* **Apps** → Miniaplicaciones autónomas con su propia interfaz de usuario y ciclo de vida independientes[cite: 1].
* **Personas** → Perfiles de comportamiento y personalidades de IA instalables[cite: 1].
* **Widgets** → Componentes interactivos de interfaz para el panel de la Control Room[cite: 1].
* **Themes** → Paquetes de CSS personalizado y estilos para la interfaz de usuario[cite: 1].
* **Skill Packs** → Paquetes de comandos slash (`/`) adicionales para la interfaz de chat[cite: 1].
* **MCP Servers** → Puentes universales de herramientas externas a través del Model Context Protocol[cite: 1].

La IA genera comandos estructurados que son interpretados y ejecutados a través del sistema de plugins[cite: 1].

## ⚡ Inicio Rápido (Instalación One-Click)
La forma más sencilla de instalar y configurar Hecos desde cero es utilizar el **Asistente de Configuración Universal**[cite: 1].

### 1. Clonar el repositorio
```bash
git clone [https://github.com/Hecos-Project/Hecos.git](https://github.com/Hecos-Project/Hecos.git)
cd Hecos
```[cite: 1]

### 2. Lanzar el Asistente de Configuración
Ejecuta el script de inicio para tu plataforma. Esto comprobará automáticamente Python, instalará las dependencias e iniciará el asistente de configuración en tu navegador[cite: 1].

**Windows:**
```powershell
.\START_SETUP_HERE_WIN.bat
```[cite: 1]

**Linux:**
```bash
bash START_SETUP_HERE_LINUX.sh
```[cite: 1]

### 3. Componentes Manuales y Scripts de Utilidad
Si prefieres gestionar los componentes por separado o realizar tareas de mantenimiento manual, utiliza estos scripts dedicados[cite: 1]:

| Plataforma | Script | Descripción |
| :--- | :--- | :--- |
| **Todas** | `main.py` | Inicia el sistema completo (Tray + WebUI + Backend) |
| **Windows** | `RESTART_TRAY_ICON_WIN.bat` | Restaurar el icono de bandeja si se cerró |
| **Linux** | `RESTART_TRAY_ICON_LINUX.sh` | Restaurar el icono de bandeja si se cerró |
| **Windows** | `scripts\windows\run\HECOS_WEB_RUN_WIN.bat` | Inicia SOLO la Interfaz Web y el Servidor |
| **Linux** | `scripts/linux/run/hecos_web_run.sh` | Inicia SOLO la Interfaz Web y el Servidor |
| **Windows** | `scripts\windows\run\HECOS_CONSOLE_RUN_WIN.bat` | Inicia SOLO la Consola de Terminal (TUI) |
| **Linux** | `scripts/linux/run/HECOS_CONSOLE_RUN.sh` | Inicia SOLO la Consola de Terminal (TUI) |
| **Windows** | `scripts\windows\setup\INSTALL_HECOS_WIN.bat` | Instalación manual de dependencias y Piper |
| **Linux** | `scripts/linux/setup/INSTALL_HECOS_LINUX.sh` | Instalación manual de dependencias y Piper |[cite: 1]

### 4. Configuración y Primera Ejecución
Hecos está diseñado para una experiencia profesional de "descargar y jugar"[cite: 1].
- En tu primera ejecución, el sistema detectará que faltan los archivos `system.yaml` y `routing_overrides.yaml`[cite: 1].
- **Generará automáticamente** estos archivos copiando las plantillas desde `hecos/config/data/*.example`[cite: 1].
- Encontrarás tu configuración personal en `hecos/config/data/system.yaml` (ajustes principales) y `routing_overrides.yaml` (reglas de enrutamiento de IA)[cite: 1].
- **Consejo profesional**: Usa el [Editor de Enrutamiento] integrado en la WebUI para modificar de forma segura estas reglas sin tocar el código[cite: 1].

### 🔐 Login & Autenticación
Hecos requiere autenticación obligatoria[cite: 1]. El inicio de sesión predeterminado por primera vez es:
- **Username:** `admin`
- **Password:** `hecos`[cite: 1]

Recomendamos encarecidamente cambiar la contraseña de inmediato desde la pestaña **Usuarios** dentro del Panel de Configuración[cite: 1].

**Recuperación de Contraseña:**
Si te quedas fuera, ejecuta `python scripts/reset_admin.py` desde la terminal para forzar una nueva contraseña, o elimina manualmente el archivo `memory/users.db` para restablecer los valores predeterminados del sistema[cite: 1].

### 🛡️ Modo Silencioso (Sin Ventanas)
Si desea que Hecos se ejecute completamente en segundo plano sin ventanas de terminal visibles[cite: 1]:
1. **Usar el Icono de Bandeja**: Inicie Hecos a través del icono de la barra de sistema. Gestionará los componentes del sistema de forma invisible en segundo plano[cite: 1].
2. **Inicio Silencioso**: Use `START_HECOS_SILENT_WIN.vbs` para un inicio 100% invisible (sin ventanas de consola)[cite: 1].
3. **Recuperación Manual**: Si cierra accidentalmente el icono, use `START_HECOS_TRAY_WIN.bat`[cite: 1].

## 🛠️ Requisitos del Sistema Esenciales (Windows)
Si acabas de reinstalar Windows o estás configurando Hecos por primera vez, debes instalar estos paquetes de sistema **fundamentales** para que todos los módulos funcionen correctamente[cite: 1]:

1. ⚙️ **Microsoft Visual C++ Redistributable (Obligatorio)**
   - *Para qué sirve*: Requerido por el motor de Memoria RAG (ONNX/FastEmbed). Sin este paquete recibirás errores de DLL faltantes y la búsqueda de documentos no funcionará[cite: 1].
   - *Descarga*: 👉 [Descargar VC++ Redist x64](https://aka.ms/vs/17/release/vc_redist.x64.exe)[cite: 1]

2. 🎵 **VLC Media Player 64-bit (Obligatorio)**
   - *Para qué sirve*: El motor de Audio y el Reproductor de Medios integrado de Hecos utilizan las librerías de VLC en segundo plano para reproducir música, alarmas y salida de voz TTS[cite: 1].
   - *Descarga*: 👉 [Descargar VLC 64-bit](https://www.videolan.org/vlc/download-windows.html)[cite: 1]

3. 👁️ **Tesseract OCR (Recomendado para la Visión)**
   - *Para qué sirve*: Necesario para las capacidades visuales avanzadas y para leer texto en la pantalla a través de OCR (`pytesseract`)[cite: 1].
   - *Descarga*: 👉 [Descargar Tesseract OCR para Windows](https://github.com/UB-Mannheim/tesseract/wiki)[cite: 1]

4. 🟢 **Node.js (Recomendado / Requerido para el desarrollo y compilación del Canvas)**
   - *Para qué sirve*: Necesario para compilar y gestionar las dependencias del módulo del Editor Visual de Flujos (ReactFlow/Vite). Si necesitas recompilar el frontend del canvas (`npm run build`), se requiere Node.js[cite: 1].
   - *Descarga*: 👉 [Descargar Node.js LTS](https://nodejs.org/)[cite: 1]

---

### 📦 Instalación Offline (Carpeta `dependencies`)
Para tu comodidad, todos los paquetes de instalación necesarios están incluidos offline directamente en la carpeta `dependencies/` en la raíz del proyecto[cite: 1]:
* `dependencies/VC_redist.x64.exe` -> Microsoft Visual C++ Redistributable (ONNX/RAG)[cite: 1]
* `dependencies/node-v24.16.0-x64.msi` -> Node.js LTS (Canvas / Frontend Build)[cite: 1]
* `dependencies/tesseract-ocr-w64-setup-5.5.0.20241111.exe` -> Tesseract OCR (Visión)[cite: 1]

*Note: Recomendamos encarecidamente instalar estos componentes antes de iniciar la configuración automática de Hecos.*[cite: 1]

## 🧠 Backends de IA Soportados (Motores LLM)

Hecos está completamente fuera de línea por defecto y requiere un motor de IA local para procesar lógica y conversación[cite: 1]. Durante la configuración, debes instalar uno de los backends independientes a continuación[cite: 1]. Hecos los detectará automáticamente[cite: 1].

### 🔹 1. Ollama (Recomendado)
Fácil de usar, rápido y optimizado para ejecutarse localmente como servicio en segundo plano[cite: 1].
- **Descarga**: 👉 https://ollama.com/download[cite: 1]
- **Configuración**: Once instalado, abre tu terminal/símbolo del sistema y ejecuta `ollama run llama3.2` para descargar y probar un modelo ligero y rápido[cite: 1]. Hecos lo detectará al instante[cite: 1].

### 🔹 2. KoboldCpp (Alternativa)
Perfecto para modelos manuales GGUF y hardware más antiguo sin grandes instalaciones[cite: 1].
- **Descarga**: 👉 https://github.com/LostRuins/koboldcpp/releases[cite: 1]
- **Configuración**: Descarga el archivo `.exe` (o el binario de Linux), haz doble clic, selecciona cualquier modelo de instrucciones GGUF descargado de HuggingFace y ejecútalo[cite: 1]. Hecos se conectará a través del puerto `5001`[cite: 1].

## 🔌 Sistema de Plugins
Hecos utiliza una arquitectura dinámica[cite: 1]. Cada plugin puede registrar comandos, ejecutar acciones del sistema y extender las capacidades de la IA[cite: 1].

Plugins incluidos:
* **Control del sistema y Gestor de archivos**[cite: 1]
* **Automatización Web y Dashboard de hardware**[cite: 1]
* **Control multimedia y Cambio de modelo**[cite: 1]
* **Gestión de memoria**[cite: 1]

Puedes instalar nuevos paquetes dinámicamente arrastrando y soltando archivos `.hpkg` en el **Package Manager** dentro de la WebUI[cite: 1]. Estos paquetes independientes contienen su propia lógica, paneles de interfaz de usuario y esquemas de configuración, haciendo que Hecos sea infinita y universalmente extensible[cite: 1]. Todos los paquetes se verifican mediante firmas digitales Ed25519 para una seguridad absoluta[cite: 1].
