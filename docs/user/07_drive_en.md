# 📁 Hecos Drive

> *"An integrated file explorer and code editor. Your AI has direct access to your workspace."*
> — Hecos WebUI Tour

**Hecos Drive** is one of the five main pillars of the Hecos WebUI. It is an integrated file manager and code editor that gives both you and the AI direct, contextual access to files in your workspace.

![Hecos - Drive](https://github.com/Hecos-Project/Hecos-Assets/blob/main////////015_Hecos_Drive_FileManager.png?raw=true)


## What is Hecos Drive?

Hecos Drive is not just a file browser — it is the bridge between the AI and your filesystem. When Hecos needs to read, write, or edit a file during a task, it uses the Drive as its workspace. You can follow along in real time.

![Hecos - Drive ](https://github.com/Hecos-Project/Hecos-Assets/blob/main/////////016_Hecos_Drive_FileManager_002.png?raw=true)


## Key Features

### 📂 File Explorer
- Browse folders and files inside your configured workspace.
- Create, rename, move, and delete files directly from the interface.
- Drag files into the Chat to send them to Hecos for analysis.

### ✏️ Integrated Code Editor
- **Syntax highlighting** for 50+ programming languages.
- **Quick navigation** between open files.
- **Live preview** for Markdown files.
- Supports direct editing of Hecos configuration files (e.g., flows, personas).

### 🤖 AI-Aware Workspace
- When you ask Hecos to *"create a Python script"* or *"edit the config file"*, it writes directly to the Drive.
- You can open any AI-generated file immediately in the editor to review or tweak it.

## How to Access It

The Drive is accessible from the **Central Hub** navigation (top menu → Drive tab) or via direct command:
```
/drive

```
